=== checklist pro prenos mezi servery ===

== priprava ==

[ ] vytvorit db usera a databazi
[ ] nasadit backend - spadne na absenci configu
[ ] prenest config (config.yml a database.yml)
[ ] nasadit znova - melo by projit, jen nenajde passenger app
[ ] vytvorit nginx config
[ ] nginx -t
[ ] restartovat nginx
[ ] overit funkcnost backendu requestem, zatim s ignorovanim certifikatu (http --verify=no)

[ ] zkompilovat frontend s env vary ukazujici na novou domenu frontendu i backendu
[ ] nasadit frontend, nginx uz by mel servovat, zatim ale bude komunikovat s backendem na starem serveru
- stranka by mela najet (treba pouzit domenu vazanou k serveru, nikoli skupine, tedy napr.
  production-tth, nikoli production-a), ale bude hlasit nevalidni certifikat

== samotny prenos ==

[ ] dotazat se backendu na starem serveru na posledni transakci
[ ] vypnout vhost na starem serveru
[ ] dumpnout db, prenest na novy server
[ ] pustit migrace
[ ] dotazat se backendu na novem serveru na posledni transakci, overit, ze odpovida
[ ] prenastavit domenu

== finale ==

[ ] pockat az resolvne na novou ip
[ ] pridat na novem serveru hostnamy do certifikatu
[ ] overit funkcnost frontendu vcetne komunikace s backendem
