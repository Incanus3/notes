problem: overriding shouldUpdate() in a DslQStoreTableLoaderBase subclass (with the intention to change the base condition, currently in QStoreTableLoaderBase) completely skips the additional logic added in DslQSTLB.

the current paths are:

QSTLB (basic, overridable condition) -> concrete subclass - overriding is safe

QSTLB (basic, overridable condition) -> DslQSTLB (calls super, but adds more logic) -> concrete subclass
- would like to override the condition defined in QSTLB, but keep the additional logic from DslQSTLB

the problem here is that .shouldUpdate() currently severs as both
- the template method that subclasses can override to customize the base condition
- the method that contains all the base logic that shouldn't be overridden
  - there is no additional base logic in QSTLB, that's why this wasn't a problem until now
  - but there is now additional base logic in DslQSTLB

what we probably need to do:
- keep the template method separate from the "base logic" method
- how do we name these though?
- what are their properties?
  - neither needs to be public
  - the "base logic" one:
    - should be overridable only by abstract middle classes, not by leaf classes
      - this currently isn't a problem, because middle classes can do a `final override` and the base one in QSTLB is currently empty
